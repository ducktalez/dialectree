# Dialectree – Meme Catalog

Memes are visual shorthand for complex argumentative situations. Each meme template
encodes a recognisable pattern — a fallacy, a discussion dynamic, or a rhetorical move.
This catalog maps meme templates to their argumentative concepts (referencing taxonomy.md).

> **Purpose:** This file serves three goals:
> 1. **Classification tool** — Users can "tag" an argument chain with a meme, instantly communicating the pattern.
> 2. **Visualisation mode** — Argument chains can be rendered as meme sequences (KI-generated).
> 3. **Engagement layer** — Memes are fun. Fun drives participation. Participation drives data quality.

> **Maintainer note:** When adding a new meme, fill in all columns. The `Taxonomy Ref`
> column must point to one or more sections in `taxonomy.md`.

## Meme Sequence Generation (Future Feature)

An argument chain can be automatically rendered as a meme sequence:

```
Argument chain:
  1. "Migration is economically necessary"           (Claim)
  2. "We have a labour shortage"                     (Reason)
  3. "Healthcare would collapse without migrants"    (Example)
  4. "So migration is important"                     (Implication)
  5. Opponent: "I don't think migration is important" (Denial)

→ Meme sequence: Patrick Star's Wallet
  Panel 1: "Is this your labour shortage?"          — "Yes"
  Panel 2: "And healthcare needs workers?"          — "Yes"
  Panel 3: "And migrants fill those jobs?"          — "Yes"
  Panel 4: "So migration is important?"             — "No"
```

### Requirements for Meme Generation

| Requirement | Description |
|------------|-------------|
| **Template Library** | Each meme template needs: image/layout specification, panel count, text placement rules |
| **Concept Mapping** | Each argumentative pattern maps to one or more meme templates (this catalog) |
| **Argument Chain Detection** | KI identifies which pattern an argument chain matches |
| **Text Fitting** | Argument text must be condensed to fit meme panel constraints (KI summarisation) |
| **User Selection** | Users can choose which meme template to apply, or accept KI suggestion |
| **Sharability** | Generated memes should be downloadable / shareable (engagement driver) |

### Implementation Phases

| Phase | Scope |
|-------|-------|
| **1. Catalog** | This document — manually curated meme-to-concept mappings |
| **2. Tagging** | Users can tag argument chains with a meme template name (like any other tag) |
| **3. Preview** | Static meme preview: select template + argument chain → generated image |
| **4. KI Suggestion** | System suggests matching meme template based on argument pattern |
| **5. Auto-Generation** | Full meme generation from argument chain (KI text fitting + image composition) |

## How to Add a New Meme

1. Identify the meme template and its standard usage.
2. Ask: *"What argumentative pattern does this encode?"*
3. Find the matching concept(s) in `taxonomy.md`.
4. Add a row to the appropriate category table above.
5. If no category fits, create a new category section.
6. If the meme maps to a concept not yet in the taxonomy, add the concept there first.
